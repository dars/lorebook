# Design: creation-choices-2024

## Context

建角流程現況：背景加值由 `_autofillBonus()` 依職業主屬性自動指派（`_bonusMode`/`_plus2`/`_plus1` 狀態與 `1/1/1` 計算分支已存在，僅無 UI）；技能步驟已有「職業選 N + 背景固定」兩區；`SpeciesOption` 有 `size`/`darkvision`/`hpPerLevel` 等機制欄位，特性其餘為顯示文字。力竭對話框全文走 `entryCatalogProvider('condition')`（內容庫 2014）。

## Goals / Non-Goals

**Goals:**
- 背景加值符合 2024 玩家自選規則（含 +1/+1/+1 模式）
- 給熟練的種族特性（人類/精靈）實際反映在角色技能上
- 人類體型可選；力竭全文與摘要同為 2024

**Non-Goals:**
- 給法術/抗性的種族特性（龍裔血統、提夫林傳承、精靈血統法術）——涉及傷害抗性與種族法術欄位，另案
- 起源專長機制效果（仍僅存名）
- 其他狀態（力竭以外）的 2014 全文覆寫——差異小，維持內容庫
- 自訂背景（背景外屬性組合）

## Decisions

### D1. 背景加值 UI：就地改造既有說明卡
`_bonusExplain()` 卡片改為互動區：頂部 +2/+1 ↔ +1/+1/+1 模式切換（SegmentedButton 樣式沿用能力值方式切換），+2/+1 模式下背景三屬性各為一列，每列可點選指派「+2」「+1」「—」（同屬性不可同時 +2/+1；指派互斥自動對調，與標準陣列下拉同哲學）。預設值維持 `_autofillBonus()` 建議。
**捨棄方案**：獨立新步驟——加值只有三屬性兩枚硬幣，不值得一步。

### D2. 種族技能以資料驅動
`SpeciesOption` 新增 `skillPickCount: int`（人類 1、精靈 1、其餘 0）與 `skillPickFrom: List<String>`（人類 = 全 18、精靈 = 洞察/感知/求生）。技能步驟在職業區之後渲染「種族（選 N）」區，重用 `_SkillRow`；與職業/背景已選重複的選項顯示鎖定（已熟練者不可再選，避免浪費）。建立時併入 `profSkills`。
**理由**：與既有 `hitDie`/`skillChoices` 資料驅動慣例一致，之後補其他種族零成本。

### D3. 人類體型：`sizeChoices` 欄位 + 基本步驟切換
`SpeciesOption` 新增 `sizeChoices: List<String>`（預設單值 = 現行 `size`；人類 `['Medium','Small']`）。基本步驟選了多體型種族時，種族說明卡下顯示體型 chips；建立時寫入所選。切換種族時重置為該種族預設。

### D4. 力竭全文本地覆寫
`_showEffectDialog` 對 `nameEn == 'Exhaustion'` 直接渲染本地 2024 全文常數（自行撰寫的規則描述，非抄書文字），不打內容庫；其餘狀態維持現行。文字與摘要同源於 2024 機制事實：每級所有 d20 檢定 −2、速度每級 −5 呎、等級 0–6、6 級死亡、長休移除 1 級、效果不疊加取當前等級。

## Risks / Trade-offs

- [加值指派 UI 增加建角認知負擔] → 預設建議值不變，不動它的玩家體驗與現況相同
- [種族技能區在精靈+特定職業組合下三選項可能全被佔用] → 洞察/感知/求生同時被職業+背景拿滿的組合存在（如遊俠+守衛）；此時鎖定項不可選會卡步驟——規則本身允許重複熟練無效益但可選；實作為：已熟練項仍可選（計入種族選擇數），顯示「已由職業/背景取得」註記
- [力竭本地文案與內容庫其餘狀態風格不一致] → 接受；2024 力竭本就只有短短幾行
