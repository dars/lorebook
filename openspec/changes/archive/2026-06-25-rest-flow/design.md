## Context

`enhance-decision-status` 與 `class-resources` 已讓長/短休按鈕直接執行（長休清臨時 HP + 回滿資源、短休回滿短休資源）。本次把休息做成正式流程：**長休加確認、短休改為資訊型對話框**。本「簡單版」聚焦核心恢復與資訊呈現，**不做生命骰擲骰自動化**。

## Goals / Non-Goals

**Goals:**
- **長休**：點擊 → 確認對話框 → 確認後完整恢復（HP 全滿、法術位全滿、職業資源全滿、臨時 HP 清空、力竭 −1）。
- **短休**：點擊 → bottom sheet，**告知**可做的事：
  - **生命骰細節**（依角色職業骰面 + 等級，如「3d6」）——僅資訊呈現。
  - **奧術恢復**（若該角色有此特性）：用**與法術相同的可展開卡（EntryCard）**呈現，點開看敘述。
  - 「完成短休」動作：回復短休類職業資源。
- 共用既有元件（EntryCard）與可變角色狀態 Notifier。

**Non-Goals:**
- 生命骰的**擲骰/花費自動化**與**長休回復一半生命骰**（玩家自行擲、用 HP +/- 調整；之後可再做）。
- 奧術恢復的**實際回補法術位邏輯**（本次只呈現敘述供參考）。
- 每 24 小時僅一次長休、休息中斷規則。
- 各職業「休息才用」能力的完整 catalog（屬靜態資料、後續）。

## Decisions

### 1. `longRest()` 擴充為完整恢復
notifier `longRest()` 由「臨時 HP 0 + 資源回滿」擴充為：`currentHp = maxHp`、所有法術位 `used = 0`、職業資源 `current = max`、`tempHp = 0`、`exhaustionLevel = max(0, exhaustionLevel - 1)`。

### 2. 長休確認對話框
點長休 → `AlertDialog`「確定長休？」（避免誤觸）→ 確認才呼叫 `longRest()`。

### 3. 短休 bottom sheet（資訊型）
點短休 → `showModalBottomSheet`：
- **生命骰**：顯示「生命骰　{level}d{faces}」。骰面由職業對應（見決策 4），數量 = 角色等級。**僅資訊**，不提供擲骰。
- **奧術恢復**：若角色 `features` 含 Arcane Recovery（以 `nameEn == 'Arcane Recovery'` 或 `name == '奧術恢復'` 偵測），以 **EntryCard** 呈現（badge 如「復」、可展開看 `description`）。
- **完成短休** 按鈕：呼叫 `shortRest()`（回滿短休資源）後關閉。

### 4. 生命骰骰面對應（本機 map，無模型變更）
職業 → 生命骰骰面：野蠻人 d12；戰士/聖騎士/遊俠 d10；吟遊詩人/牧師/德魯伊/武僧/盜賊/術士/邪術師/法師 → d6 或 d8（依 5.5e：法師/術士 d6、其餘多為 d8）。以 className 對應，數量 = `level`。本次**僅顯示**，不追蹤已花用 → 無需新增 `Character` 欄位。

### 5. 重用 EntryCard 呈現休息能力
奧術恢復等「休息才用」能力沿用 `EntryCard`（與法術一致的可展開卡），標籤標示其性質；之後若有更多休息能力可同樣呈現。

## Risks / Trade-offs

- **[生命骰僅資訊、不自動擲]** → 簡化首版；玩家自行擲骰並用 HP +/- 調整。日後可加「花生命骰 + 擲骰回血」與「長休回半」。
- **[奧術恢復僅呈現敘述]** → 不自動回補法術位；玩家自行依敘述操作（法術位本就可點調整）。實際自動化留後續。
- **[奧術恢復偵測靠特性名稱]** → 以 feature 名稱判定，屬啟發式；之後靜態資料可改為明確標記「休息能力」。
