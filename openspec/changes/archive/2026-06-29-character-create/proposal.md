## Why

角色選擇頁目前無法新增角色（「新增角色（簡化版）」只有 spec、無實作、頁面也無入口）。需要一個能實際建立可用角色卡的流程，作為使用 app 的起點。

## What Changes

- **引導式建角流程**（依 D&D 2024 規則，面向非專業玩家；六步）：
  1. **基本**：角色圖佔位 + 名稱 + 陣營 + **種族**（選項 + 敘述/重點）；等級固定 1。
  2. **職業**：選項列出，**選取顯示敘述（1~2 百字）+ 重點提示**。
  3. **背景**：同上（敘述 + 重點）。
  4. **能力值**：可切換 **標準陣列／購點／擲骰** 三種方式（**預設標準陣列並套用建議配置**）；標準陣列與購點以**值下拉**直接選（購點顯示每值點數成本與剩餘點數、買不起變灰），擲骰**手動填入**；**背景加值**移至頁首說明（已套用 +2/+1）；各方式有對應正確性檢查。
  5. **技能與熟練**：技能熟練（職業選 N + 背景固定 2）；豁免熟練由職業自動帶（唯讀）。
  6. **確認**：預覽摘要 → 建立。
- **每個選項顯示說明**：固定顯示**敘述**（風味 + 白話）＋**重點提示**（機制條列）。敘述為**文案佔位**（最終文案由文案人員撰寫，不在本變更範圍）；重點提示由機制資料生成。
- **角色圖佔位**（基本步驟頂部）：未上傳圖片時顯示**剪影人像 + 問號**，採漩渦深淵風格底。
- **六角屬性圖（radar）**：顯示於**能力值頁與確認頁**（呈現實際分數形狀，是它真正有意義之處）；種族/職業/背景頁以重點提示文字說明機制即可。
- **衍生數值自動計算**：修正值、熟練加值（Lv1=+2）、HP（生命骰max+體質）、AC（10+敏捷）、先攻、被動察覺、施法 DC/命中、生命骰骰面；豁免/技能依熟練。
- **入口與導向**：角色選擇頁加「新增角色」入口 → 流程 → 建立後加入名冊、設為當前角色、進入主畫面。
- **目錄（機制資料齊全 + 敘述佔位）**：種族表（速度、體型、感官/黑暗視覺、特性）；職業表（生命骰、施法屬性、固定豁免 2、可選技能 + 數量）；背景表（能力候選 3、固定技能 2、起源專長名）。各表含 `敘述`（佔位）與 `重點`（由機制生成）。

## Impact

- **程式碼**：`features/character/presentation/character_create_page.dart` + 各步驟 widget（引導式流程）、`character_select_page.dart`（新增入口）、`character_providers.dart`（沿用 `CharacterListNotifier.add` + 選取）、`app/router.dart`（建角後設定選取/導航）；新增種族/職業/背景/技能目錄常數。
- **能力**：character-management「新增角色」擴充為引導式流程（種族/職業/背景各頁含說明、2024 背景能力加值、職業豁免、技能熟練）。
- **範圍界線**：**敘述文案的最終內容**（由文案人員撰寫，本變更僅建欄位 + 佔位）；起源專長的機制效果、起始裝備/金幣、法術、語言、工具熟練、子職（2024 Lv3）、Point Buy、擲骰、Lv>1、種族特性的完整機制化（抗性等）→ **後續**。目錄先涵蓋**精選常見清單**（約 8~10 種族、12 職業、8~10 背景），其餘漸增。跨 session 持久化（Supabase）後續。
- **相依套件**：不新增。
