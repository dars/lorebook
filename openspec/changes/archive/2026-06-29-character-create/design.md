## Context

`CharacterSelectPage` 列出 `characterListProvider`，但無新增。`Character`（freezed）必填僅 `id/name/abilityScores`，其餘有預設（含 `AbilityScore.proficientSave`、技能清單）。目標使用者為**非專業玩家**，故採引導式流程、每個選擇都附說明。依 D&D 2024 規則建出機制正確的角色 → 組 `Character` → 加入名冊。沿用既有 mock/記憶體模式。

## Goals / Non-Goals

**Goals:**
- 引導式流程（基本 / 種族 / 職業 / 背景 / 能力值 / 技能 / 確認），每個種族·職業·背景選項固定顯示**敘述 + 重點提示**。
- 依 2024：標準陣列 + 背景能力加值；職業固定豁免；技能熟練（職業選 + 背景固定）；種族設速度/體型/感官。
- 衍生數值自動算；建立後加入名冊並設為當前角色。

**Non-Goals:**
- 敘述**文案最終內容**（文案人員負責；本變更放佔位）。
- 起源專長機制效果、起始裝備/金幣、法術、語言、工具、種族特性完整機制化（抗性等）。
- 子職（2024 Lv3）、Point Buy、擲骰、Lv>1、跨 session 持久化。

## Decisions

### 1. 流程容器與步驟
`CharacterCreatePage`（ConsumerStatefulWidget）以 step index 管理**六步**，底部「上一步 / 下一步 / 建立」；本地 state 收集輸入；經角色選擇頁推入（root navigator 全螢幕）。
1. **基本**：頂部角色圖佔位 + 名稱（必填）+ 陣營 + **種族**（選項 + 敘述/重點）；等級固定 1。
2. **職業**、3. **背景**：各一頁，共用「選擇頁」版型（見 2）。
4. **能力值**（見 3）、5. **技能與熟練**（見 4）、6. **確認**（見 5）。
- **六角屬性圖只在「有真實數值」處顯示**：能力值頁與確認頁（種族/職業/背景頁不放，靠重點提示文字說明即可）。

### 2. 種族 / 職業 / 背景 選擇頁（共用版型）
- 選項列（精選清單）；選取者高亮（金色）。
- 選取後**固定顯示**：
  - **敘述**：1~2 百字風味 + 白話（資料來源為目錄的 `description` 欄位，**佔位文案**）。
  - **重點提示**：機制條列（由目錄機制欄位生成），例：職業「生命骰 d12 · 豁免 力/體 · 主屬性 力量 · 可選技能 …」；背景「能力 力/敏/體 · 技能 運動/威嚇 · 專長 X」；種族「速度 30 · 黑暗視覺 · 特性 …」。
- 未選不可續行。

### 3. 能力值（三種方式切換 + 背景加值 + 六角圖）
- **頂部方式切換**：標準陣列／購點／擲骰（segmented）。**預設標準陣列並套用建議配置**（autofill，依「職業主屬性 → 體質/敏捷 → 其餘」）。
- **背景加值**移至**頁首說明**（顯示已套用的 +2/+1，例「士兵：力量 +2、體質 +1」）；固定（由建議帶入），不在此頁編輯。
- 統一列格式：`能力名　[控制]　→　最終值 (修正值)`；最終值 = 基礎值 + 背景加值（上限 20），修正值 `floor((score-10)/2)`。
- **標準陣列**：每能力一個**值下拉**（列出 6 個值並標明目前在誰身上，選已佔用者→對調）；上方顯示「剩餘可分配 N／6」+「套用建議／清空」。完成（皆指派、為 6 值之排列）方可續行。
- **購點**：每能力一個**值下拉**（8–15，列出各值累計點數，剩餘點數買不起者變灰）；上方顯示**點數條 + 剩餘／27** + 重置（回全 8）；可重複值；恆 ≤27 點。
- **擲骰**：每能力一個**數字輸入框**（自行擲 4d6 去最低 ×6 填入，App 不代擲）；合理夾 3–18、可重複、可 <8；皆填妥方可續行。
- 本頁與確認頁顯示**六角屬性圖**（實際分數正規化）。

### 4. 技能與熟練
- **技能熟練**：職業可選清單選 N 項 + 背景固定 2 項（自動勾選、唯讀）。
- **豁免熟練**：職業固定 2 項，唯讀顯示。
- 選滿職業技能數才可續。

### 5. 確認 + 建立（角色預覽）
完整預覽建好的角色卡，由上而下：
1. **頭部**：頭像佔位（圓形漩渦）+ 名稱 + 種族·職業·等級·背景·陣營。
2. **戰鬥數值**（置頂）：HP / AC / 先攻 / 被動察覺 / 熟練（皆當參考值）；施法職業另顯示 施法DC / 施法命中。
3. **六角屬性圖**（實際分數）。
4. **屬性值**：六能力卡（最終分數 + 修正值）。
5. **豁免**：六項含加值（熟練金色），以 `CollapsibleSection` **預設收起**（summary 顯示「6 項」）。
6. **技能**：18 項含加值（熟練金色），`CollapsibleSection` **預設收起**（summary「18 項」）。
- 底部「建立」→ 組 `Character`（含 `proficientSave`、技能熟練、速度等）→ `characterListProvider.add()` → 設 `selectedCharacterIdProvider` → 導航 `/main/decision`。

### 6. 衍生數值
- `proficiencyBonus = 2`；`modifier = floor((score-10)/2)`（背景加值後）。
- `maxHp = currentHp = 生命骰max + conMod`；`ac = 10 + dexMod`；`initiative = dexMod`；`passivePerception = 10 + wisMod`。
- 豁免/技能 = 能力修正 +（熟練則 +pb）。施法職業：`spellDc = 8+pb+abilMod`、`spellAttack = pb+abilMod`。
- `hitDieFaces`、`speed`（種族）、法術/裝備清單空。

### 6b. 角色圖佔位（基本步驟）
名稱步驟頂部顯示角色圖區；未上傳時顯示**佔位**：剪影人像 + 問號，漩渦深淵風格底（漸層 + 同心/漩渦感）。為日後圖片上傳預留位置（本變更僅佔位，不含上傳）。

### 6c. 六角屬性圖（radar）
共用元件（`CustomPainter` 六角雷達）。只在**有真實數值**處顯示——**能力值頁與確認頁**，以實際分數（含背景加值）正規化繪製真正形狀（種族/職業/背景頁不放，避免顯示無意義的投射圖；機制以重點提示文字說明）。
- 未來可將同元件放進角色卡的屬性頁當常駐元件（後續）。

### 7. 目錄常數（機制齊全 + 敘述佔位）
- **種族**：`{ cn, en, speed, size, darkvision?, traits:[...], description(佔位) }`。
- **職業** × 12：`{ cn, en, hitDie, spellAbility?, saves:[2], primaryAbilities:[...]（六角圖高亮用）, skillChoices:{count, from:[...]}, description(佔位) }`。
- **背景**：`{ cn, en, abilities:[3], skills:[2], originFeat(名稱), description(佔位) }`。
- **技能**：沿用既有技能定義（中英 + 對應能力）。
- **重點提示**由上述機制欄位生成；`description` 為佔位，待文案。

## Risks / Trade-offs

- **[步驟變多]** → 由 4 步增為 7 步（引導式）；換得非專業玩家友善、每步有說明。原「中間版」因「固定顯示說明」需求自然升級為引導式。
- **[內容量]** → 種族/職業/背景敘述為佔位（文案外包）；機制資料先做精選常見清單，其餘漸增。
- **[種族機制簡化]** → 僅設速度/體型/感官 + 記錄特性文字；抗性等完整機制化後續。
- **[仍非完整]** → 無專長效果/裝備/法術；建出角色可玩但相關頁仍空，待後續。
