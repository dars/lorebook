> **起始提案** — 以下為依「動作經濟分類 / 分組排序 / 視覺密度」三軸提出的方向，細節（尤其視覺）待討論或 Pencil 對齊後再定案。

## Why

戰鬥當下最常看的就是「我這回合能做什麼」，但目前行動頁把 **ACTION 動作 / BONUS ACTION 附贈動作 / REACTION 反應** 做成三個分離的 SectionTitle 區塊，加上動作內部（攻擊／施法／其他）分組與卡片樣式不夠一致，掃視成本偏高、動作經濟的歸屬也不夠一眼分明。

## What Changes

- **動作經濟分類更清楚**
  - 將 動作 / 附贈動作 / 反應 整合於單一「動作」大區，內部以**動作經濟分節**呈現，每節標頭清楚標示類別（動作／附贈動作／反應）與一句提示（如「每回合 1 個」「觸發時」）。
  - 讓「這回合的動作 / 附贈 / 反應分別有什麼」一眼分類清楚。
- **分組與排序**
  - 動作內依戰鬥優先序排列：**攻擊 → 施法（戲法、法術依環）→ 其他動作**（Dash/Disengage/Help…）。
  - 統一卡片：所有可執行項目都用 `EntryCard`（其他動作改用，移除與舊 `AbilityCard` 混用）。
- **視覺/密度**
  - 統一卡片樣式與間距、經濟分節標頭更輕量、減少重複的大標題，提升資訊密度與一致性。
- **可收合**
  - 新增共用 `CollapsibleSection`，**所有頂層區段**（狀態/資源/移動/動作/附贈動作/反應/檢定/休息）皆可點標題收合。
  - 動作 / 附贈動作 / 反應 維持頂層同階；動作內類別（攻擊/施法/其他）另可收合，收合顯示計數；動作/附贈/反應 收合時顯示內容摘要。
  - 預設：頂層全展開、動作內攻擊/施法展開、其他收合。解決高階施法者動作清單過長。

## Capabilities

### Modified Capabilities
- `decision`: 將 Action / Bonus Action / Reaction 三項需求整併為單一「動作（含動作經濟分節）」呈現，並統一分組排序與卡片樣式。

## Impact

- **程式碼**：`features/decision/presentation/sections/` 的 `action_section.dart`、`bonus_action_section.dart`、`reaction_section.dart` 重構/整併（可能合成單一 `actions_section.dart`）；統一改用 `EntryCard`、新增輕量「動作經濟分節標頭」widget。
- **資料層**：不變（沿用現有 weapons / spells / cantrips；動作經濟分類由 UI 判定，如法術 castingTime 判附贈/反應）。
- **版型**：手機與平板皆受影響，沿用同一份 widget。
- **相依套件**：不新增。
- **純顯示調整**：不改資料模型、不影響其他頁面。
