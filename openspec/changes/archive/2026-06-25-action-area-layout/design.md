## Context

行動頁目前有三個分離的 section：`ActionSection`（攻擊/施法/其他）、`BonusActionSection`、`ReactionSection`，各自一個 `SectionTitle`。卡片混用 `EntryCard`（武器/法術）與 `AbilityCard`（其他動作）。本次以「動作經濟分類 / 分組排序 / 視覺密度」重整呈現，**純顯示層**，不動資料模型。

## Goals / Non-Goals

**Goals:**
- 動作 / 附贈動作 / 反應維持為**三個頂層 section**（與狀態/移動同階），但各自可收合。
- 「動作」section 內分組排序：攻擊 → 施法（戲法/各環）→ 其他動作，分層清楚。
- 統一卡片樣式（全用 `EntryCard`）與間距，提升密度與一致性。

**Non-Goals:**
- 資料模型變更、動作的實際執行邏輯（仍只是顯示/追蹤既有資料）。
- 完整動作經濟規則引擎（哪些法術算附贈/反應，沿用 castingTime 簡單判定）。

## Decisions

### 1. 動作經濟＝三個頂層 section（同階），動作內再分層
`actions_section.dart` 統一管理共享的收合狀態，但**輸出三個頂層 section**（動作 / 附贈動作 / 反應），與 STATUS / MOVEMENT 同階、用相同的 `SectionTitle` 樣式（英中標題＋分隔線）＋ chevron。**不**再外包一層「ACTION」總標題。
```
…（頂層）狀態 / 資源 / 移動
ACTION 動作（頂層 section，可收合）
  ├─ 攻擊（類別，可收合）：武器 EntryCard…
  ├─ 施法（類別，可收合；無施法則整段不顯示）
  │    ├─ 戲法（無戲法則略過）
  │    └─ 一環 / 二環 / 三環 …（依角色可用環數）
  ├─ 職業動作（類別）：花費動作的職業招式（如武僧某些動作）
  └─ 其他（類別）：Dodge/Disengage/Help…
BONUS ACTION 附贈動作（頂層 section，可收合）：狂暴、連擊、重振旗鼓、附贈施法…（無則「無可用」）
REACTION 反應（頂層 section，可收合）：護盾術、機會攻擊、偏斜箭矢、瞬避…（無則「無可用」）
CHECKS 檢定 / REST 休息（頂層）…
```
- **理由**：動作經濟本就是與狀態/移動並列的頂層資訊；「動作」內部才需要 類別 → 施法子層 的分層。標頭權重：頂層 section 標題 > 類別小標 > 環數細標。

### 1c. 職業獨特動作（非施法職業也有戲）
職業招式（狂暴、連擊、重振旗鼓、偏斜箭矢、狡詐動作…）依其**動作經濟**落在對應 L1 分節：
- 多數為**附贈動作 / 反應** → 列於對應分節（野蠻人/武僧/盜賊在此才有招牌動作）。
- 少數花費**動作**者 → 動作分節的「L2 職業動作」類別。
- 資料來源：需角色的「職業動作」清單（名稱、經濟、簡述）；mock 先帶代表性招式（如野蠻人狂暴、武僧連擊）。**此為純顯示，唯需要一份職業動作資料**（細節於 tasks/apply 評估，可能小幅擴充 mock）。

### 1b. 施法子分層為資料驅動（涵蓋所有職業/子職）
「施法」底下不為各職業寫死，由角色資料推導：
- **戲法**：角色有戲法才顯示（聖騎/遊俠等半施法者無戲法 → 略過）。
- **各環**：依角色實際可用環數列出（全施法 1–9、半施法 1–5、三分之一 1–4、邪術師契約位）。
- **無施法職業**（野蠻人、基礎武僧）→ 不出現「施法」類別。
- 邪術師祕法奧義（6–9）依先前決議放法術清單標「1/天」，不在此另列。

### 2. 三層標頭，權重遞減
- **L1 動作經濟分節標頭** `_EconomyHeader`：icon + 類別名 + 一句提示（最醒目）。
- **L2 類別小標**（攻擊/施法/其他）：次級權重。
- **L3 環數細標**（戲法/一環/二環…）：最輕，作為施法內的群組分隔。
- 三層字級/顏色/縮排明顯區隔，讓階層一眼可辨。

### 3. 統一卡片為 `EntryCard`
- 其他動作（Dash/Disengage/Help/Hide/Ready/Search/Use Object）改用 `EntryCard`（badge 如「動」，可附簡短規則描述供展開；無描述則不展開）。
- 移除 `AbilityCard` 在這些 section 的使用（`AbilityCard`/`SubSectionHeader` 若他處未用可後續清理）。

### 4. 分組排序
- L2 類別固定序：**攻擊 → 施法 → 其他**。
- L3 施法內固定序：**戲法 → 一環 → 二環 → …**（環數遞增）。
- 附贈/反應：依現有判定（castingTime 含 bonus/reaction、特定能力）。

### 5. 視覺/密度
- 統一卡片間距（如 8）、經濟分節間距（如 16）、標頭字級；移除重複大標題。手機平板共用，必要時 layout 層微調。

### 6. 可收合（所有頂層區段 + 動作內類別）
- **共用元件 `CollapsibleSection`**（置於 `app/theme/decorations.dart`）：標題樣式對齊 `SectionTitle` + chevron，點標題切換收合；自管展開狀態（StatefulWidget），可選 `summary`（收合時顯示）。
- **所有頂層區段皆可收合**：狀態 / 資源 / 移動 / 動作 / 附贈動作 / 反應 / 檢定 / 休息 各自用 `CollapsibleSection` 包裝（每個 section widget 自包，Resources 仍可在無資源時整段隱藏）。
- **動作內類別**（攻擊/施法/其他）另有自己的收合（金色小標 + chevron + 計數），狀態由 `ActionsSection` 管理。
- **施法子層**（戲法/各環）再下一層也可收合，且**預設收合**（只露標頭+計數）；點環才展開該層法術。狀態由 `ActionsSection` 的 `_expandedRings` 管理（預設空＝全收合）。
- **收合摘要**：動作/附贈/反應收合時顯示內容摘要；其他頂層區段不顯示摘要。
- **預設狀態**：所有頂層區段展開；動作內「攻擊/施法」展開、「其他」收合。
- 收合狀態本階段為前端 UI 狀態（不持久化；之後可記住）。
- **理由**：動作清單在高階施法者會很長；讓整頁每段都能收合，玩家可把目光集中在當下要用的段落。

## Risks / Trade-offs

- **[整併三 section 的工作量]** → 合成單一 `actions_section`，但邏輯沿用現有（武器/戲法/法術/附贈/反應的取得方式照舊搬移），降低風險。
- **[其他動作的規則描述]** → 改 `EntryCard` 後可附簡短規則（Dash「移動加倍」等）；描述文字屬靜態資料，先以精簡常數帶入。
- **[視覺細節未定]** → 本提案先定結構與分組；確切間距/標頭樣式待 Pencil 對齊後微調。
