## MODIFIED Requirements

### Requirement: 情境式頁首
App SHALL 依分頁是否綁定當前角色，顯示對應的頁首：角色情境頁顯示共用角色頭區塊（僅呈現當前角色資訊，不含切換入口），全域/系統頁顯示純標題頁首。

每個導航分頁以 `characterScoped` 旗標標記是否屬於角色情境。角色切換入口改置於設定頁（見 system 能力）。

#### Scenario: 角色情境頁顯示角色頭
- **WHEN** 使用者在角色情境分頁（行動 / 角色 / 旅程）
- **THEN** 頂部顯示角色徽章 + 角色名稱 + 種族·職業 + Level 徽章
- **THEN** 不顯示下拉箭頭，頁首僅為當前角色的資訊呈現

#### Scenario: 全域頁顯示純標題頁首
- **WHEN** 使用者在全域/系統分頁（設定）
- **THEN** 頂部顯示純標題頁首（`PageHeader`，含分頁名稱），不顯示角色資訊
