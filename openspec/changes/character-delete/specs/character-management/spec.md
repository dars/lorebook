# character-management Delta

## MODIFIED Requirements

### Requirement: 刪除角色
App SHALL 允許使用者刪除自己的角色：於角色選擇畫面長按角色卡觸發，經確認後執行；已登入時以雲端軟刪除（tombstone）為準、成功方移除本地清單，確保跨裝置一致。

#### Scenario: 長按觸發與確認
- **WHEN** 使用者於角色選擇畫面長按角色卡
- **THEN** 顯示確認對話框，內容含角色名稱與「此操作無法復原」提示
- **WHEN** 使用者點「取消」
- **THEN** 關閉對話框，角色不受影響

#### Scenario: 確認刪除（已登入）
- **WHEN** 使用者於確認對話框點「刪除」
- **THEN** 先對雲端執行軟刪除（`deleted_at` tombstone）
- **THEN** 雲端成功後自本地清單移除該角色，清單即時刷新
- **THEN** 刪除進行中對話框按鈕顯示忙碌狀態且不可重複觸發

#### Scenario: 雲端刪除失敗
- **WHEN** 已登入但雲端軟刪除失敗（如網路中斷）
- **THEN** 顯示「刪除失敗」提示，本地清單不變（避免下次同步時角色復活）

#### Scenario: 刪除當前選取角色
- **WHEN** 被刪除的角色為當前選取角色
- **THEN** 清除選取狀態並停留在角色選擇畫面

#### Scenario: 未登入（離線模式）刪除
- **WHEN** 未登入狀態下確認刪除
- **THEN** 僅自本地清單移除（無雲端同步）

#### Scenario: 刪除至清單為空
- **WHEN** 刪除後清單無任何角色
- **THEN** 顯示既有的空狀態與新增角色引導
