# character-management — 升級入口（delta）

## ADDED Requirements

### Requirement: 升級入口
角色頁頁首右上角的 LEVEL 徽章（共用 `CharacterHeader`）SHALL 於角色頁作為升級入口：點擊後彈出確認對話框詢問是否調升等級（含目標等級，如「調升至 Lv 4？」），使用者確認後進入升級流程、取消則關閉；其他顯示該頁首的頁面（如行動頁）維持純顯示不可點擊。等級 ≥ 3 且子職為空且內容庫可用時，對話框另提供「補選子職」選項。升級完成返回後，頁首與各分頁即時反映新數值。

#### Scenario: 點擊 LEVEL 徽章詢問升級
- **WHEN** 使用者於角色頁點擊頁首右上角 LEVEL 徽章且角色等級 < 20
- **THEN** 彈出確認對話框「調升至 Lv N？」（N 為現等級 +1）
- **WHEN** 使用者確認
- **THEN** 進入引導式升級流程
- **WHEN** 使用者取消
- **THEN** 關閉對話框，角色不受影響

#### Scenario: 等級上限
- **WHEN** 角色等級為 20 且使用者點擊 LEVEL 徽章
- **THEN** 顯示已達最高等級的提示，不進入升級流程

#### Scenario: 非角色頁不可觸發
- **WHEN** LEVEL 徽章顯示於角色頁以外的頁面（如行動頁頁首）
- **THEN** 徽章為純顯示，點擊不觸發升級

#### Scenario: 補選子職
- **WHEN** 角色等級 ≥ 3 且子職為空且內容庫可用
- **AND** 使用者點擊 LEVEL 徽章
- **THEN** 確認對話框另提供「補選子職」選項，點擊進入單步子職選擇

#### Scenario: 升級後即時反映
- **WHEN** 升級流程完成並返回角色頁
- **THEN** 頁首 LEVEL 徽章與總覽、屬性、法術等分頁即時顯示升級後數值
